Multiclass Classification

MNIST handwritten digit classification using a feedforward neural network.

In this example, we’ll train a network to recognize handwritten digits. We’ll first use a direct feedforward network and then design a convolutional neural network in another example to see how the two compare.

We start by importing the required modules.

use neuro::data::{DataSetError, ImageDataSet};
use neuro::models::Network;
use neuro::losses;
use neuro::activations::Activation;
use neuro::metrics::Metrics;
use neuro::optimizers::Adam;
use neuro::layers::Dense;
use neuro::tensor::*;
use neuro::regularizers::Regularizer;

use std::path::Path;

Then, the main function is created. We choose to return a Result so that we can use the ? shortcut notation and avoid several match statements.

fn main() -> Result<(), DataSetError> {

The data are then loaded in an ImageDataSet which is created from a Path object. The path we give to from_path must contain a subfolder named train in which each training sample is stored in a directory containing the name of the class. That is, for the MNIST dataset, we have the following architecture:

    0/
      img1.jpg
      img2.jpg
      ...
    1/

      img2.jpg
      ...
    2/

      img2.jpg
      ...
    ...

The root directory can optionally contain a test subfolder used for testing purpose. The method from_path takes two additional arguments: the size of the images and the fraction of the train set used for validation. In this case, 10% of the images will be used for validation. All images will be resized to be squared with the given size. We then print some information on the dataset.

// Create the dataset
let path = Path::new("datasets/MNIST");
let data = ImageDataSet::from_path(&path, 28, 0.1)?;
println!("{}", data);

This code will print:

Output shape: 	 [10 1 1]
Number of training samples: 	 37800
Number of validation samples: 	 4200
Number of classes: 	 10

The next step is to create a network and add some layers. For this example, we use the softmax cross-entropy loss, the Adam optimizer with a learning rate of 0.003, the accuracy as metrics, and no regularizer. Two dense layers with 32 and 10 units respectively are added to the network. The hidden layer uses a ReLU activation function and the output layer a softmax activation function. We keep the default weights and biases initializers which are uniform distribution with He scaling for the weights and zeros for the biases. Note that in neuro, the softmax cross-entropy loss function requires that the last layer uses a softmax activation function. Also note that the dense layer will automatically reshape the input to be one-dimensional. Some information on the network are then printed.

// Create the neural network
let mut nn = Network::new(&data, losses::SoftmaxCrossEntropy, Adam::new(0.003), Some(vec![Metrics::Accuracy]), None);
nn.add(Dense::new(32, Activation::ReLU));
nn.add(Dense::new(10, Activation::Softmax));

println!("{}", nn);

Calling println will output:

Layer 		 Parameters
------------------------
Dense 		 25120
Dense 		 330

The network is trained for 10 epochs using a batch size of 128 and the training progress is displayed at each epoch.

// Fit the network
nn.fit(128, 10, Some(1));

The first line tells us that the library uses the GPU of my computer which supports OpenCL. We then see the training loss, validation loss, and accuracy for each epoch. After a single epoch, the network already performs quite well with an accuracy of 93.99%. At the end of the 10 epochs, the network is able to classify the images with 96.03% accuracy. This is pretty good considering the simple architecture that we’ve used. Looking at the output, we see that the best accuracy was reached after 5 epochs, with an accuracy of 96.70%. If we look at the training and validation losses for the last epochs, we see that the training loss keeps decreasing but the validation loss stays around 0.13. This indicates that the network is starting to overfit and its ability to generalize is deteriorating. In order to prevent this to happen, we can add some regularization to the network. One way to do it is by adding L2 regularization to the loss function. That is, the loss function is augmented by the sum of the L2-norm of the weights of each layer. The regularization is added by modifying the last argument of the command used previously:

let mut nn = Network::new(&data, losses::SoftmaxCrossEntropy, Adam::new(0.003), Some(vec![Metrics::Accuracy]), Some(Regularizer::L2(1e-4)));

where we have chosen for the weight of the regularization. The network is then trained again for 10 epochs:

Once the network has been trained, it’s time to make some predictions on unseen samples. We load a few images using the load_img_vec method provided by ImageDataSet, predict the output class for each of them, and print the prediction along with the confidence of the network (probability). Finally, we return Ok(()) at the end of the function to exit cleanly if everything went fine.

// Predict output on test images
let input = data.load_img_vec(&vec![
  Path::new("datasets/MNIST/test/2/img_1.jpg"),
  Path::new("datasets/MNIST/test/1/img_18.jpg"),
  Path::new("datasets/MNIST/test/0/img_9.jpg"),
  Path::new("datasets/MNIST/test/3/img_10.jpg")
])?;

let predictions = nn.predict_class(&input);
print_prediction(&predictions);

Ok(())
}

fn print_prediction(predictions: &Vec<(String, PrimitiveType)>) {
    println!("Predictions:");
    let mut index = 0;
    for (class, probability) in predictions {
        index += 1;
        println!("image {}: class: {}, probability: {}", index, class, probability);
    }
}
